# TypeScript Adapter

The TypeScript adapter is the default (and only) adapter for v0.3.0. It emits
ESM TypeScript source files, test files, and config files for the target
project using `bun`-compatible patterns.

## File conventions

| Kind | Directory | Extension |
|------|-----------|-----------|
| `source` | `<repo>/tools/` | `.ts` |
| `test` | `<repo>/tests/` | `.test.ts` |
| `config` | `<repo>/` (root) | `.ts` |
| `tool-implementation` | `<repo>/tools/` | `.ts` |

## Module system

All emitted TypeScript MUST use **ESM** (`import`/`export` syntax). Do not use
CommonJS (`require`, `module.exports`). Use named exports, not default exports.

## Import conventions

Use `bun`-compatible imports. For filesystem operations, use Bun's built-in
`Bun.file()` and `Bun.write()` APIs instead of `node:fs`:

```typescript
// Preferred — bun-native
const file = Bun.file("path/to/file");
const content = await file.text();

// Preferred — bun-native write
await Bun.write("path/to/file", content);

// Avoid — node: prefix imports (not needed in bun)
// import { readFileSync } from "node:fs";
```

For standard library features where bun's native API isn't available, use
plain import paths (no `node:` prefix):

```typescript
import { resolve, join } from "path";
// not import { resolve } from "node:path"
```

## Zod validation (tools)

Every emitted agent tool that accepts input should use Zod for runtime
validation:

```typescript
import { z } from "zod";

const RestGetInput = z.object({
  url: z.string().url(),
  headers: z.record(z.string()).optional(),
});

export type RestGetInput = z.infer<typeof RestGetInput>;
```

## `assertSandbox(...)` pattern

Every emitted function that accesses the filesystem MUST include an
`assertSandbox(path)` call at the top. This is a runtime sandbox guard that
checks the target path is within the allowed paths from
`constraints_inherited.sandbox.allowed_paths`.

### Template for `assertSandbox` helper

Emit this helper at the top of every source file that accesses the filesystem:

```typescript
/**
 * Runtime sandbox guard.
 * Throws if `targetPath` is not within any allowed path from constraints.
 * @param targetPath - The path the function intends to access
 * @throws {Error} If the path violates sandbox constraints
 */
function assertSandbox(targetPath: string): void {
  // ALLOWED_PATHS comes from constraints_inherited.sandbox.allowed_paths
  // hydated from the manifest at emit time
  const ALLOWED_PATHS: string[] = ["src", "tests", "tools"];
  const allowed = ALLOWED_PATHS.some((p) => targetPath.startsWith(p));
  if (!allowed) {
    throw new Error(
      `Sandbox violation: path "${targetPath}" is not in allowed paths [${ALLOWED_PATHS.join(", ")}]`,
    );
  }
}
```

If `constraints_inherited` is missing or empty, the helper is omitted.

## Export pattern

Each source module must export the functions listed in `manifest.exports[]`:

| manifest.exports[].name | exported binding |
|------------------------|-----------------|
| `"restGet"` | `export function restGet(...)` |
| `"search"` | `export function search(...)` |
| `"VERSION"` | `export const VERSION = "0.3.0"` |

Each exported function MUST have a JSDoc comment with the `description` from
the manifest (or the file's `intent` if no per-export description exists).

```typescript
/**
 * REST GET wrapper — auto-generated by harness-emit-code
 * @remarks Derived from manifest file intent: "REST GET wrapper for the public API"
 */
export function restGet(input: unknown): unknown {
  assertSandbox("read_only");
  // TODO: implement REST GET wrapper
  return input;
}
```

### tool-implementation: `tools/rest-get.ts`

```typescript
/**
 * REST GET wrapper — auto-generated by harness-emit-code
 * Makes HTTP GET requests with sandbox enforcement.
 */
import { z } from "zod";

// --- Sandbox enforcement ---

const ALLOWED_PATHS: string[] = ["src", "tests", "tools"];

function assertSandbox(mode: string, operation: string): void {
  // mode can be "read_only" or "write"
  // Operation names are informational
  if (mode === "read_only") {
    // Read-only operations are always allowed within the sandbox
    return;
  }
  throw new Error(`Sandbox violation: write mode not allowed for ${operation}`);
}

/**
 * REST GET wrapper — auto-generated by harness-emit-code
 */
export function restGet(input: unknown): unknown {
  assertSandbox("read_only", "restGet");
  // TODO: implement REST GET wrapper
  return input;
}

export const VERSION = "0.3.0";
```

### middleware helper: `middleware/assert-sandbox.ts`

```typescript
/**
 * Centralized sandbox enforcement — auto-generated by harness-emit-code.
 * Imported by tool modules that need filesystem access.
 */
export function assertSandbox(mode: string, operation: string): void {
  if (mode === "read_only") {
    return;
  }
  throw new Error(`Sandbox violation: write mode not allowed for ${operation}`);
}
```

## Validation

After emitting all files:

1. **Tier 1 (per-file):** Confirm file exists, is non-empty, and exports match
   the manifest declaration.
2. **Tier 2 (project-level):** Run `tsc --noEmit` on the target project root
   and expect exit code 0.
